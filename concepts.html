Part 1 — Language Concepts

HTML

HyperText Markup Language

Concept What it means Example Used in your portfolio
Tag A keyword in angle brackets that tells the browser what something is <p>Hello</p> Every element: nav, section, div, h1…
Attribute Extra info added inside a tag to configure it <a href="..."> href, class, id, target
Semantic Tags Tags with meaning — tell the browser the purpose of content <nav> <section> <footer> nav, section, footer used throughout
Heading tags h1 is biggest/most important, h6 is smallest. Only one h1 per page <h1> to <h6> h1 for name, h2 for section titles, h3 for project names
div & span Generic containers. div = block (takes full width), span = inline (fits content) <div class="card"> Cards, grids, stat boxes, badges
class vs id class = reusable (many elements). id = unique (one element per page) class="card" id="hero" id for sections (#hero, #about), class for styling groups
Anchor tag Creates links — to other pages, sections on same page, or email <a href="#about">About</a> Nav links, project links, contact email, social buttons
meta tags Info about the page — for browsers and search engines. Goes in <head> <meta charset="UTF-8"> charset and viewport at top of file
SVG Scalable Vector Graphics — crisp icons and shapes drawn with code, not images <svg><path d="..."/></svg> GitHub, LinkedIn, arrow icons throughout
CSS

Cascading Style Sheets

Concept What it means Example Used in your portfolio
Selector Picks which HTML element(s) to style. Can target tag, class, or id nav { } .card { } #hero { } Every style rule in your CSS
CSS Variables Store values once, reuse everywhere. Defined in :root. Change one → updates everywhere --accent: #6c63ff; color: var(--accent); All colours defined as --bg, --accent, --muted etc.
Box Model Every element is a box: content → padding → border → margin padding: 1rem; margin: 0 auto; Spacing on every card, section, button
Flexbox Arranges items in a row or column. Great for nav bars, button groups, centered content display: flex; align-items: center; gap: 1rem; Nav, hero CTA buttons, social links, step items
Grid Two-dimensional layout. Define columns and rows. Best for card grids display: grid; grid-template-columns: 1fr 1fr; Skills grid, projects grid, about grid, stats grid
Position Controls where an element sits. fixed = stays on scroll, absolute = relative to parent position: fixed; top: 0; Nav bar (fixed), glow blobs (absolute), WIP badge (absolute)
Pseudo-classes Style an element in a specific state — hover, focus, first-child etc. .btn:hover { background: #7c74ff; } All hover effects on buttons, cards, links
Pseudo-elements ::before and ::after insert virtual content before/after an element .hero-tag::before { content: ''; } Pulsing dot on hero tag, noise texture overlay on body
@keyframes Defines a CSS animation — what happens at 0%, 50%, 100% of the animation @keyframes fadeUp { from { opacity:0; } to { opacity:1; } } fadeUp (hero text), pulse (green dot)
transition Smoothly animates a property change (like hover colour change) transition: color 0.2s ease; All hover effects — buttons, cards, nav links
Media queries Apply different styles at different screen sizes — makes site mobile-friendly @media (max-width: 768px) { nav { padding: 1rem; } } Mobile responsive section at bottom of CSS
clamp() Sets a value between a min and max — fluid font sizing that adapts to screen font-size: clamp(3rem, 7vw, 5.5rem); h1 and section titles — responsive without media queries
backdrop-filter Applies a visual effect (like blur) to what's behind an element backdrop-filter: blur(16px); Navbar glass effect when scrolling over content
JS

JavaScript

Concept What it means Example Used in your portfolio
const / let Declare variables. const = never reassigned. let = can change later const name = "Ertaza"; let count = 0; const observer = new IntersectionObserver(...)
Arrow function Short way to write a function using =>. No need for the "function" keyword const greet = (name) => "Hi " + name; Callback inside IntersectionObserver
querySelectorAll Finds all HTML elements matching a CSS selector. Returns a list document.querySelectorAll('.card') Selects all .reveal elements to observe
forEach Loops through every item in a list and runs a function on each items.forEach(item => console.log(item)) Loops through each .reveal element to attach observer
classList Add, remove, or toggle CSS classes on an element with JavaScript el.classList.add('visible') Adds 'visible' class to trigger CSS animation
IntersectionObserver Watches if an element enters or exits the visible screen area (viewport) new IntersectionObserver(callback, { threshold: 0.1 }) Triggers scroll reveal animations as user scrolls down
Callback function A function passed into another function to be called later observer.observe(el) → runs callback when el is visible The entries => { } function inside IntersectionObserver
Conditional (if) Run code only when a condition is true if (e.isIntersecting) { e.target.classList.add('visible') } Only adds class when element is on screen

Phase A

One-Time Setup (Do This Once on Your Laptop)

  • 1

    Install Git

    Download and install Git from the official site. Keep all default settings during install.

    # Download from:
    https://git-scm.com/download/win

    # After install, verify it works — open VS Code terminal and type:
    git --version
    You should see something like: git version 2.x.x. If yes, Git is installed!
  • 2

    Tell Git who you are

    Git needs your name and email to label your commits. Do this once — it applies to all projects.

    git config --global user.name "Hafiz Muhammad Ertaza"
    git config --global user.email "muhammadertaza620@gmail.com"
    Use the same email as your GitHub account.
Phase B

Create the Repository on GitHub

  • 3

    Create a new repository on GitHub

    Go to github.com while logged in as M-Ertaza1 and create a new repo.

    # On GitHub.com:
    Click the + button (top right) → New repository

    Repository name: portfolio-website
    Description: Personal developer portfolio — HTML, CSS, JavaScript
    Visibility: Public

    # IMPORTANT: Do NOT tick "Add README" — leave it empty
    Click Create repository
    Do not initialise with README, .gitignore, or license. Keep it completely empty — you will push your files into it.
  • 4

    Copy your repository URL

    After creating, GitHub shows a page with setup instructions. Copy the HTTPS URL.

    # Your repo URL will look like this:
    https://github.com/M-Ertaza1/portfolio-website.git

    # Copy it — you'll need it in the next step
Phase C

Push Your Files from VS Code Terminal

  • 5

    Open your portfolio folder in VS Code terminal

    In VS Code, open the terminal: press Ctrl + ` (backtick key, below Esc). Make sure you are inside the folder that contains index.html.

    # Check you're in the right folder — should show index.html:
    dir

    # If not, navigate to your folder:
    cd C:\Users\YourName\Desktop\portfolio-website
    In VS Code you can also right-click your folder in the Explorer panel → "Open in Integrated Terminal" to open directly in the right folder.
  • 6

    Initialise Git in your folder

    This turns your folder into a Git repository. Run this only once per project.

    git init
    You'll see: "Initialized empty Git repository in ..." — that's correct.
  • 7

    Stage your files

    Tell Git which files to include in your first save. The dot ( . ) means "all files".

    git add .

    # Verify what was staged:
    git status
    You should see index.html listed in green under "Changes to be committed".
  • 8

    Make your first commit

    A commit is like saving a snapshot of your project with a description of what you did.

    git commit -m "Initial commit: Add portfolio website"
    The message in quotes describes this snapshot. Always write something meaningful.
  • 9

    Connect to your GitHub repository

    Link your local folder to the GitHub repo you created in Step 3.

    git remote add origin https://github.com/M-Ertaza1/portfolio-website.git

    # Verify the connection:
    git remote -v
    remote -v should show: origin https://github.com/M-Ertaza1/portfolio-website.git (fetch) and (push).
  • 10

    Rename branch to main (if needed)

    GitHub uses "main" as the default branch. Make sure your local branch matches.

    git branch -M main
  • 11

    Push to GitHub 🚀

    Upload your files to GitHub. GitHub may ask you to log in — use your GitHub username and password (or a personal access token).

    git push -u origin main
    After this, go to github.com/M-Ertaza1/portfolio-website — your index.html should be there!
    If GitHub asks for a password and rejects it, you need a Personal Access Token (PAT). Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Generate new token (classic) → tick "repo" → copy and use as your password.
Phase D

Every Time You Make Changes (After First Push)

  • ★

    The 3-command routine — memorise this

    Every time you update your portfolio or any project, run these 3 commands in order:

    # 1. Stage your changes
    git add .

    # 2. Commit with a description of what you changed
    git commit -m "Update: add LinkedIn link and fix email"

    # 3. Push to GitHub
    git push origin main
    Good commit messages describe WHAT you changed. Bad: "update". Good: "Fix nav links on mobile, update email in contact section".
Phase E

Make Your Portfolio Live on the Internet (Free Hosting)

  • 12

    Enable GitHub Pages

    GitHub can host your HTML site for free — no server needed. Your site will be live at a real URL.

    # On GitHub.com:
    Go to your repo: github.com/M-Ertaza1/portfolio-website
    Click Settings tab (top of repo)
    Scroll down to Pages (left sidebar)
    Under Source: select Deploy from a branch
    Branch: main → folder: / (root)
    Click Save

    # Wait 1-2 minutes, then your site is live at:
    https://M-Ertaza1.github.io/portfolio-website
    Share this URL on your CV, LinkedIn, and GitHub profile README. This is your live portfolio link!
